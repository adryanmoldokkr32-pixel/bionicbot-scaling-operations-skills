# Sovereign Data Geofencing

This skill focuses on 'Legal Privacy Architecture', ensuring that sensitive assets never touch servers in 'High-Risk' or 'Over-Regulated' jurisdictions.

## Instructions
1. Map the physical locations of all cloud provider nodes (AWS, Vercel, Hetzner).
2. Implement routing rules that block data transfer to specific country codes.
3. Set up automated encryption triggers if data is forced to transit through unsafe nodes.
4. Audit system logs to verify 100% compliance with geofencing rules.

## Examples
- "Ensure all trade logs stay on Swiss or Romanian servers and never transit through the US."
